# Analysis Rubric

这份参考文件用于细化“七维分析”时的判断标准。只有在需要更深入分析时再读取，不必每次完整展开。

## 使用方式

先按场景对齐，再选最 relevant 的维度展开。不要机械地把每个小点都写进最终报告。

## 1. 功能

关注页面到底支持用户完成什么，以及能力的完整度和差异化。

观察点：

- 是否有核心任务入口
- 是否有辅助决策信息
- 是否有筛选、收藏、分享、客服、评价、推荐等辅助功能
- 是否出现差异化组件，如订阅、达人背书、快捷下单、套装推荐

可追问：

- 谁的功能覆盖更完整
- 谁在关键节点减少了用户决策成本
- 谁更强调单次转化，谁更强调长期留存

## 2. 价格

关注价格如何被展示、解释和锚定。

观察点：

- 券后价、原价、划线价、会员价是否同时出现
- 是否突出满减、限时折扣、红包、返现、赠品
- 价格是否位于视觉焦点
- 是否通过组合套餐、对比价、最低价承诺做锚定

可追问：

- 谁更强调“便宜”
- 谁更强调“值”
- 谁通过复杂优惠提高吸引力，但也增加理解成本

## 3. 用户体验

关注用户完成目标的路径长度、理解成本和交互阻力。

观察点：

- 一屏是否能看懂主要价值
- CTA 是否清晰
- 需要多少步才能到下一关键动作
- 是否存在过多弹窗、表单、切换、跳转
- 是否提供即时反馈、默认选项、推荐选项

可追问：

- 谁更快引导用户进入下一步
- 谁更稳妥但更重
- 是否存在明显流失点

## 4. 设计风格

关注视觉语言如何服务转化与品牌感知。

观察点：

- 色彩体系是否强调促销、品质、社区或科技感
- 信息密度高低
- 视觉重心是否稳定
- 字体层级、卡片样式、图文比例是否统一

可追问：

- 风格更偏效率还是品牌感
- 页面是在“压缩决策”还是“丰富解释”
- 是否存在视觉噪音或层级混乱

## 5. 增长策略

关注从界面上能反推出哪些促转化手段。

观察点：

- 倒计时、限量、限时、稀缺性
- 销量、评价、达人、好友在买、榜单等社会认同
- 邀请、裂变、分享、返利、补贴
- 直播、短视频、内容种草到交易的衔接

可追问：

- 它更依赖内容驱动还是货架驱动
- 它是在做即时成交还是持续培育
- 它如何降低首次尝试门槛

## 6. 商业化模式

关注页面透露出的收入逻辑和平台取舍。

观察点：

- 佣金、广告、会员、增值服务、套餐升级等痕迹
- 是否引导商家付费、达人合作、流量采买
- 是否优先展示高客单或高转化商品

可追问：

- 页面设计更服务用户，还是更服务平台收入
- 它是否通过推荐机制偏向高商业价值内容

## 7. 优劣势

这一维是前六维的总结，而不是重复。

输出时尽量写成：

- 优点：为什么有效
- 缺点：成本或风险是什么
- 机会点：如果我们借鉴，最适合切入哪里
- 风险点：照搬后可能带来的副作用

## 置信度标记建议

- 高：截图证据直接支持结论
- 中：有明确视觉信号，但仍包含少量推断
- 低：信息不完整，只能给方向性判断

当价格、规则、功能文案看不清时，默认把相关结论降为中或低，不要伪装成确定事实。
